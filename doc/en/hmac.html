Class:jCastle.hmac#

해시 기반 메세지 인증 코드(Hash-based Message Authentication Code, HMac)는 해쉬 함수를 기반으로 하는 정보의 무결성을 검증하는 메시지 인증 코드입니다.

new jCastle.hmac([hash_name])#

jCastle.hmac 오브젝트를 생성합니다.

  • hash_name <string> 생성된 객체에 사용할 알고리듬 이름.
    이 변수에 null값이 올 수 있습니다. 만일 null값이 주어지면 jCastle.hmac::start 혹은 jCastle.hmac::digest에 주어지는 변수 값에 algoName이라는 프로퍼티와 그 값이 주어져야 합니다.

Static method: jCastle.hmac.create([hash_name])#

jCastle.hmac 오브젝트를 생성합니다. new jCastle.hmac()와 동일합니다.

  • hash_name <string>생성된 객체에 사용할 해쉬 알고리듬 이름.
  • Return Value: jCastle.hmac 오브젝트를 생성하여 리턴합니다.
var hmac = jCastle.hmac.create('skein-1024');
hmac.start({
  key: key, 
  outputBits: 512
});
hmac.update(message);
var code = hmac.finalize();

// 위의 코드는 다음과 같이 사용할 수 있습니다.
var hmac = jCastle.hmac.create('skein-1024/512'); // outputBits property can be here
hmac.start({
    key: key
});
hmac.update(message);
var code = hmac.finalize();

Static method: jCastle.hmac.start(options)#

jCastle.hmac 오브젝트를 생성한 후 hmac.hmac() 메소드를 실행합니다.

  • options <Object> 해쉬 MAC 코드를 생성하기 위한 변수들을 담은 오브젝트. hmac.start()를 참조하세요.
  • Return Value: jCastle.hmac 오브젝트를 생성하여 hmac.start()를 수행한 후 동일 오브젝트를 리턴합니다.

Static method: .jCastle.hmac.digest(message, options) #

주어진 데이터를 가지고 해쉬 MAC 코드를 생성하고 그 결과 값을 리턴합니다. options 오브젝트에는 반드시 algoName이 주어져야 합니다.

  • message <Buffer> 해쉬 MAC코드를 생성할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • options <Object> 해쉬 MAC 코드 생성에 필요한 변수들을 담은 오브젝트로 hmac.start의 options 변수를 참조하세요.
  • Return Value: MAC 코드를 <Buffer>로 리턴합니다.

Static method: jCastle.hmac.getOID(hash_name)#

베이스가 되는 알고리듬의 해쉬 MAC 알고리듬 OID(Object ID)가 존재할 경우 해당 OID값을 리턴합니다.

  • hash_name <string> 해쉬 MAC 알고리듬의 베이스 알고리듬 명칭.
    OID가 등록된 베이스 알고리듬은 다음과 같습니다: gost(gost3411), sha-1, sha-224, sha-256, sha-384, sha-512, md5, tiger.
  • Return Value: 해당 OID를 <string>로 리턴합니다.

Static method: jCastle.hmac.getAlgoName(hmac_algo)#

해쉬 MAC 알고리듬에서 베이스가 되는 알고리듬 명칭을 추출하여 리턴합니다.

  • hmac_algo <string> 해쉬 MAC 알고리듬 명칭.
  • Return Value: 베이스 알고리듬 명칭을 <string>로 리턴합니다.

hmac.start(options)#

해쉬 MAC 코드를 생성하기 위한 파라미터를 받아 프로세스를 준비합니다.

  • options <Object> 해쉬 MAC 코드 생성을 위한 변수들을 담은 객체
    • algoName <string> 해쉬 MAC 알고리듬의 베이스가 되는 해쉬 알고리듬 이름.
    • key|hmacKey <Buffer> 해쉬 MAC 코드 생성을 위한 키로 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다. 입력 스트링은 UTF-8 인코딩으로 간주되기 보다 binary 스트링으로 간주된다는 점을 유의하세요.
    • outputBits <Integer> 해쉬 결과 값 리턴시 스트링 길이를 지정합니다.
      outputBits의 경우 비트 길이를 의미합니다.
    • outputSize <Integer> 해쉬 결과 값 리턴시 스트링 길이를 지정합니다.
      outputSize의 경우 바이트 길이를 의미합니다.
  • Return Value: 오브젝트 자신을 리턴합니다. 
var hmac = jCastle.hmac.create('skein-1024/512'); // outputBits property can be here
hmac.start({
  key: key
});
hmac.update("The quick fox jumps");
hmac.update(" over the lazy dog!");
var code = hmac.finalize();

hmac.update(message)#

MAC 코드 생성 프로세스를 실행합니다.

  • message <Buffer> 해쉬 MAC코드를 생성할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • Return Value: 오브젝트 자신을 리턴합니다.

hmac.finalize([message])#

프로세스를 종료하고 결과를 리턴합니다.

  • message <Buffer> 해쉬 MAC코드를 생성할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • Return Value: 생성된 해쉬 MAC코드를 <Buffer>로 리턴합니다.

hmac.digest(message [, options])#

해쉬 MAC코드를 생성합니다.

  • message <Buffer> 해쉬 MAC코드를 생성할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • options <Object> MAC코드 생성에 필요한 변수들을 담은 오브젝트로 hmac.start의 options 변수를 참조하세요.
  • Return Value: 생성된 해쉬 MAC코드를 <Buffer> 데이터를 리턴합니다.
