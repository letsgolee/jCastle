Class:jCastle.mcrypt#

jCastle.mcrypt is the algorithm for message cryptography using a symmetric key.
Using a symmetric key means that the key for encryption of a message and creation of a ciphertext and the key for decryption of the ciphertext are the same each other.
The security of encryption using a symmetric key relies on the length of the key and the management of safty of the key generally. The symmetric algorithm fits on the closed network, for the sender of the ciphertext and the receiver that receive the cipertext and decrypt it both have the same key and should keep it safe from others.
To use the symmetric key for encryption and decryption on the open network, a safe exchange of the key with the recipient must first come to consideration.

mcrypt.start(options)#

Starts the process of encryption or decryption.

  • options <Object> Options object
    • algoName <string> Algorithm name for ciphergraphy work.
    • key <Buffer> Encryption/Decryption key. If it is <string> value then it is converted to <Buffer>. The input string is NOT considered as UTF-8 but binary string.
    • isEncryption <Boolean> Flag for the direction of the process. If encryption then true, otherwise false.
    • mode <string> Cipher mode for encryption. If you want to know all the mode information then refer to jCastle.mcrypt.mode.
    • padding <string> Padding name. Cipher mode like CBC needs padding process for block cipher process.
    • blockSize <Integer> Block size. If the algorithm requires a variable block size then the value should be given.
    • nonce <Buffer> Nonce value. The cipher mode that supports AEAD(Authenticated Encryption with Associated Data) need it for calculation of MAC.
      For example, CCM uses CTR mode internally and it creates iv using the given nonce value.
    • additionalData <Buffer> Additional data. A cipher mode that supports AEAD needs an associated text or an additional data. If it is <string> value then it is converted to <Buffer>.
    • tweakKey <Buffer> Tweak key. It is used in XTS mode for XOR process.
    • dataUnit <Integer> Data Unit block size. Its value is used for encryption of storage data in XTS mode.
    • sbox <string> A substitution-box name. If the algorithm is GOST then sbox can be defined. the usable sbox type is like:
      • GostR3411_94_TestParamSet(default)
      • Gost28147_TestParamSet(E-TEST)
      • Gost28147_CryptoProParamSetA(E-A)
      • Gost28147_CryptoProParamSetB(E-B)
      • Gost28147_CryptoProParamSetC(E-C)
      • Gost28147_CryptoProParamSetD(E-D)
    • ctsType <string> CTS type. It is the type in CTS mode. You can use one of CS1, CS2 and CS3(default).
  • Return Value: Returns its own instance. 
var algo_name = "AES-256";
var options = {
  key: key,
  iv: initial_vector,
  mode: "CBC",
  isEncryption: true,
  padding: "pkcs7"
};
var message = "This is the Message!";

var cipher = new jCastle.mcrypt(algo_name);
cipher.start(options);
cipher.update(message);
var ciphertext = cipher.finalize();

//--------------------------------------------------

var algo_name = "AES-256";
var options = {
  algoName: algo_name,
  key: key,
  iv: initial_vector,
  mode: "CBC",
  isEncryption: false,
  padding: "pkcs7"
};

var plaintext = new jCastle.mcrypt().start(options).update(ciphertext).finalize();

mcrypt.update(message)#

Gets the input data for encryption or decryption process.

  • message <Buffer> A message data. If it is <string> value then it is converted to <Buffer>.
  • Return Value: Returns its own instance.
var algo_name = "AES-256";
var params = {
    key: key,
  iv: initial_vector,
  mode: "CBC",
  isEncryption: true,
  padding: "pkcs7"
};
var message1 = "This is the Message!";
var message2 = " How are you?";

var cipher = new jCastle.mcrypt(algo_name);
cipher.start(params);
cipher.update(message1);
cipher.update(message2);
var ciphertext = cipher.finalize();

mcrypt.encrypt(message, options) #

Encrypts the message and returns the result of ciphertext.

  • message <Buffer> A message data. If it is <string> value then it is converted to <Buffer>.
  • options <Object> Options object. Refer to mcrypt.start()'s options parameter.
  • Return Value: Returns the result as <Buffer>.
var algo_name = "AES-256";
var message = "This is the Message!";
var options = {
  algoName: algo_name,
  key: key,
  iv: initial_vector,
  mode: "CBC",
  isEncryption: true
  padding: "pkcs7"
};

var ciphertext = new jCastle.mcrypt().encrypt(message, options);

options.isEncryption = false;
var plaintext = jCastle.mcrypt.decrypt(ciphertext, options);
