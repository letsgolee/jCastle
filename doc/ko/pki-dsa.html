Class:jCastle.pki.dsa#

DSA(Digital Signature Algorithm)는 미국 NIST(National Institute of Standards and Technology)가 개발한 디지탈 서명을 위한 알고리즘으로, 디지탈 서명과 관련하여 미국에서 처음으로 연방 정보 처리 표준(Federal Information Processing Standard, FIPS)으로 채택되었습니다. 1991년 NIST (National Institute of Standards and Technology)는 디지탈 서명의 표준(Digital Signature Standard, DSS)으로 DSA를 제안하였는데, 당시 RSA 디지탈 서명에 이미 많은 투자를 한 소프트웨어 회사들이 반대했음에도 불구하고, NIST는 1994년 FIPS 186에서 DSA를 디지탈 서명 표준안으로 채택하였습니다.

FIPS 186은 이후 4개의 개정판을 발표하였는데, FIPS 186-1 (1998년)부터 FIPS 186-4 (2013년)까지 여러 해에 걸쳐 개정되었습니다. 1998년에 발표된 FIPS 186-1 에서는 DSA와 RSA를 함께 DSS 표준으로 하였으며, 2000년에 발표된 FIPS 186-2 에서는 DSA, RSA와 더불어 ECDSA를 디지탈 서명에 사용할 수 있는 표준으로 정하였습니다. 최근 2019년에 발표된 FIPS 186-5 초안(draft)에서는 새로운 디지탈 서명을 생성하는 곳에서 더이상 DSA를 사용하지 말고 단지 이전의 DSA 서명을 검증하는 부분에만 DSA를 사용할 것을 제안하였으며, 새로운 디지탈 서명에는 타원곡선(Elliptic curve) 서명의 일종인 EdDSA(Edwards-Curve Digital Signature Algorithm)를 사용하도록 제안하고 있습니다.

DSA는 비대칭 공개키 방식으로서, DSA 디지탈 서명은 공개키와 개인키를 쌍으로 사용하는데, 개인키를 사용하여 디지탈 서명을 생성하고 공개키를 사용하여 그 서명을 검증합니다.

Static method: jCastle.pki.dsa.getPredefinedParameters(bits)#

미리 생성한 Parameters 목록 중 랜덤으로 리턴합니다.

NIST에서 미리 생성한 Parameters 목록 중 랜덤으로 리턴합니다.

dsa.generateParameters(options = {}) #

Parameters를 생성합니다.

  • otions <Object> 옵션 객체로 다음의 정보가 올 수 있습니다.
    • bits <Integer> Parameter p의 비트 크기. 기본 값은 1024입니다.
    • hashAlgo <string> 랜덤 함수에서 사용할 해쉬 함수. 주어지지 않으면 jCastle에서는 rc4를 이용합니다.
    • seed <Buffer> 랜덤 함수에 사용될 Seed값. 주어지지 않으면 jCastle에서는 랜덤으로 생성합니다.

dsa.generateKeypair([parameters]) #

공개키를 포함한 개인키를 생성합니다.

  • parameters <Object> Parameters 객체가 주어지면 이 값을 기준으로 개인키와 공개키를 생성합니다. 만일 주어지지 않으면 dsa.generateParameters()를 통해 이미 생성되어 있어야 합니다.

dsa.sign(str, options = {}) #

메시지에 대해 서명을 생성합니다.

  • str <Buffer> 서명할 스트링. 만일 스트링이 오면 UTF-8 스트링으로 인식하여 <Buffer>로 변환합니다.
  • otions <Object> 옵션 객체.
    • hashAlgo <string> 사용할 해쉬 알고리듬 명. 기본 값은 sha-1입니다.
    • returnType <string> 리턴할 서명의 형식. 올 수 있는 값은 세 종류로, <Buffer> 값인 r과 s를 concatenation하여 리턴하는 concat과, 객체형태로 리턴하는 object과, Asn#1형식으로 리턴하는 asn1이 있습니다.
  • Return Value: 메시지에 대한 서명을 생성하여 returnType 형태로 리턴합니다.

dsa.verify(str, signature, options = {}) #

메시지에 대해 생성된 서명을 검증합니다.

  • str <Buffer> 검증할 스트링. 만일 스트링이 오면 UTF-8 스트링으로 인식하여 <Buffer>로 변환합니다.
  • signature <Buffer> 검증할 서명.
  • otions <Object> 옵션 객체로 hashAlgo 값이 올 수 있습니다(기본 값은 sha-1입니다).
  • Return Value: 메시지에 대한 서명을 검증하여 맞으면 true, 그렇지 않으면 false를 <Boolean> 형태로 리턴합니다.
