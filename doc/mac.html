Class:jCastle.mac#

메시지 인증 코드(Message Authentication Code, 약칭 MAC)는 메시지의 인증에 쓰이는 작은 크기의 정보입니다.
MAC 알고리즘은 비밀 키를 입력받고, 임의-길이의 메시지를 인증합니다.
그리고 출력으로써 MAC(때때로 태그로 알려져있다)을 출력합니다.
MAC 값은 검증자(비밀 키를 소유한 사람)의 허가에 의해서 메시지의 데이터 인증과 더불어 무결성을 보호합니다.

new jCastle.mac([mac_name])#

jCastle.mac 오브젝트를 생성합니다.

  • mac_name <string> 생성된 객체에 사용할 알고리듬 이름.
    이 변수에 null값이 올 수 있습니다. 만일 null값이 주어지면 jCastle.mac::start 혹은 jCastle.mac::digest에 주어지는 변수 값에 macName이라는 프로퍼티와 그 값이 주어져야 합니다.

Static method: jCastle.mac.create([mac_name])#

jCastle.mac 오브젝트를 생성합니다. new jCastle.mac()와 동일합니다.

  • mac_name <string>생성된 객체에 사용할 맥 알고리듬 이름.
  • Return Value: jCastle.mac 오브젝트를 생성하여 리턴합니다.
var mac_name = "CMAC";
var mac = jCastle.mac.create(mac_name);
mac.start({
  key: key,
  algoName: 'aes-128'
});
mac.update("The quick fox jumps");
mac.update(" over the lazy dog!");
var code = mac.finalize().toString('hex');
console.log(code);

Static method: jCastle.mac.start(options)#

jCastle.mac 오브젝트를 생성한 후 digest.mac() 메소드를 실행합니다.

  • options <Object> MAC 코드를 생성하기 위한 변수들을 담은 오브젝트. mac.start()를 참조하세요.
  • Return Value: jCastle.mac 오브젝트를 생성하여 mac.start()를 수행한 후 동일 오브젝트를 리턴합니다.

Static method: .jCastle.mac.digest(message, options) #

주어진 데이터를 가지고 MAC코드를 생성하고 그 결과 값을 리턴합니다. options 오브젝트에는 반드시 macName이 주어져야 합니다.

  • message <Buffer> MAC코드를 생성할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • options <Object> MAC 코드 생성에 필요한 변수들을 담은 오브젝트로 mac.start의 options 변수를 참조하세요.
  • Return Value: MAC 코드를 <Buffer>로 리턴합니다.

Static method: jCastle.mac.getOID(algo_name)#

베이스가 되는 알고리듬의 MAC 알고리듬 OID(Object ID)가 존재할 경우 해당 OID값을 리턴합니다.

  • algo_name <string> MAC 알고리듬의 베이스 알고리듬 명칭.
    OID가 등록된 베이스 알고리듬은 다음과 같습니다: des, seed-128, aria-128, aria-192, aria-256, cast5, rc4.
  • Return Value: 해당 OID를 <string>로 리턴합니다.

Static method: jCastle.mac.getMacName(algo_name)#

베이스가 되는 알고리듬의 OID가 등록된 MAC 알고리듬이 존재할 경우 해당 MAC 알고리듬 명칭을 리턴합니다.

  • algo_name <string> MAC 알고리듬의 베이스 알고리듬 명칭.
  • Return Value: 해당 MAC 알고리듬 명칭을 <string>로 리턴합니다.

Static method: jCastle.mac.getAlgoName(mac_algo)#

MAC 알고리듬에서 베이스가 되는 알고리듬 명칭을 추출하여 리턴합니다.

  • mac_algo <string> MAC 알고리듬 명칭.
  • Return Value: 베이스 알고리듬 명칭을 <string>로 리턴합니다.

mac.reset() #

입력되었던 모든 데이터들을 초기화 합니다. macName은 제외됩니다.

  • Return Value: 오브젝트 자신을 리턴합니다.

mac.start(options)#

MAC 코드를 생성하기 위한 파라미터를 받아 프로세스를 준비합니다.

  • options <Object> MAC 코드 생성을 위한 변수들을 담은 객체
    • macName <string> MAC 알고리듬 이름.
      객체 생성시 알고리듬을 지정하지 않았을 경우 필요합니다.
    • algoName <string> MAC 알고리듬의 베이스가 되는 알고리듬 이름.
    • algorithm <Object> 베이스 알고리듬 객체. 만일 algoName대신 알고리듬 객체를 사용하고자 할 때
    • key|macKey <Buffer> MAC 코드 생성을 위한 키로 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다. 입력 스트링은 UTF-8 인코딩으로 간주되기 보다 binary 스트링으로 간주된다는 점을 유의하세요.
    • iv <Buffer> Initial Vector 값. 설정한 MAC 모드에서 iv값을 요구할 수 있습니다.
  • Return Value: 오브젝트 자신을 리턴합니다. 
var mac_name = "CMAC";
var mac = jCastle.mac.create(mac_name);
mac.start({
  key: key,
  algoName: 'aes-128'
});
mac.update("The quick fox jumps");
mac.update(" over the lazy dog!");
var code = mac.finalize().toString('hex');
console.log(code);

mac.update(message)#

MAC 코드 생성 프로세스를 실행합니다.

  • message <Buffer> MAC코드를 생성할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • Return Value: 오브젝트 자신을 리턴합니다.

mac.finalize([message])#

프로세스를 종료하고 결과를 리턴합니다.

  • message <Buffer> MAC코드를 생성할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • Return Value: 생성된 MAC코드를 <Buffer>로 리턴합니다.

mac.digest(message [, options])#

MAC코드를 생성합니다.

  • message <Buffer> MAC코드를 생성할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • options <Object> MAC코드 생성에 필요한 변수들을 담은 오브젝트로 mac.start의 options 변수를 참조하세요.
  • Return Value: 생성된 MAC코드를 <Buffer> 데이터를 리턴합니다.
