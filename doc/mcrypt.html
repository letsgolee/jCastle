Class:jCastle.mcrypt#

jCastle.mcrypt은 대칭키 암호화 및 복호화 알고리듬입니다.
대칭키 알고리듬이란 암호문을 생성(암호화)할 때 사용하는 키와 암호문으로부터 평문을 복원(복호화)할 때 사용하는 키가 동일한 암호화 알고리듬을 말합니다.
대칭키 암호화 알고리듬은 안전성은 키의 길이, 키의 안전한 관리에 상대적으로 의존성이 높습니다. 암호문의 작성자와 이의 수신자가 동일한 키를 비밀리에 관리해야 하므로 폐쇄적인 특성을 갖는 네트워크 상에서 적합합니다.
공개된 네트워크에서 사용하기 위해서는 수신자와 안전한 키 교환이 우선시 되어야 합니다.

new jCastle.mcrypt([algo_name])#

jCastle.mcrypt 오브젝트를 생성합니다.

  • algo_name <string>생성된 객체에 사용할 알고리듬 이름.
    이 변수에 null값이 올 수 있습니다. 만일 null값이 주어지면 jCastle.mcrypt::start 혹은 jCastle.mcrypt::encrypt에 주어지는 변수 값에 algoName이라는 프로퍼티와 그 값이 주어져야 합니다.

  • options <Object> 암호화 혹은 복호화 과정을 수행하기 위한 변수들을 담은 오브젝트. mcrypt.start()를 참조하세요.
  • Return Value: jCastle.mcrypt 오브젝트를 생성하여 mcrypt.start()를 수행한 후 동일 오브젝트를 리턴합니다.

mcrypt.start(options)#

암호화 또는 복호화 과정을 시작합니다.

  • options <Object> 옵션 객체
    • algoName <string> 메시지 암호화/복호화에 사용할 알고리즘 명칭
    • key <Buffer> 메시지를 암호화/복호화하기 위해 필요한 키로 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다. 입력 스트링은 UTF-8 인코딩으로 간주되기 보다 binary 스트링으로 간주된다는 점을 유의하세요.
    • isEncryption <Boolean> 암호화 혹은 복호화인지에 대한 Boolean 변수.
    • mode <string> 암호화/복호화에 사용되는 블럭 혹은 스트림 모드로 jCastle에서 지원하는 모드 정보는 jCastle.mcrypt.mode를 참조하세요.
    • padding <string> CBC모드처럼 블럭 모드에서 블럭처리를 위해 문자의 길이를 블럭 사이즈로 맞추기 위해 사용합니다.
    • blockSize <Integer> 암호화에 사용할 블럭 사이즈. 만일 사용하고자 하는 알고리즘이 가변적인 블럭 사이즈를 사용한다면 이 값이 주어져야 합니다.
    • nonce <Buffer> AEAD(Authenticated Encryption with Associated Data)를 지원하는 모드에서 MAC 계산을 하는데 사용됩니다.
      CCM의 경우 내부적으로 CTR모드를 사용하므로 주어진 nonce를 이용해 필요한 iv를 생성합니다.
    • additionalData <Buffer> AEAD를 지원하는 모드에서 추가적인 텍스트(associated text)혹은 데이터(additional data)가 필요합니다. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
    • tweakKey <Buffer> XTS모드에서 내부적으로 메시지 XOR하기 위한 변수
    • dataUnit <Integer> XTS모드에서 스토리지 데이터 암호화에 필요한 데이터 블록 값.
    • sbox <string> 알고리듬을 GOST로 지정시 내부적으로 사용될 sbox를 지정할 수 있습니다. 사용할 수 있는 sbox 타입은 다음과 같습니다.
      • GostR3411_94_TestParamSet(기본값)
      • Gost28147_TestParamSet(E-TEST)
      • Gost28147_CryptoProParamSetA(E-A)
      • Gost28147_CryptoProParamSetB(E-B)
      • Gost28147_CryptoProParamSetC(E-C)
      • Gost28147_CryptoProParamSetD(E-D)
    • ctsType <string> CTS모드에서 사용하는 CTS 타입. CS1, CS2, CS3(기본 값)으로 세 가지가 있습니다.
  • Return Value: 오브젝트 자신을 리턴합니다. 
var algo_name = "AES-256";
var options = {
  key: key,
  iv: initial_vector,
  mode: "CBC",
  isEncryption: true,
  padding: "pkcs7"
};
var message = "This is the Message!";

var cipher = new jCastle.mcrypt(algo_name);
cipher.start(options);
cipher.update(message);
var ciphertext = cipher.finalize();

//--------------------------------------------------

var algo_name = "AES-256";
var options = {
  algoName: algo_name,
  key: key,
  iv: initial_vector,
  mode: "CBC",
  isEncryption: false,
  padding: "pkcs7"
};

var plaintext = new jCastle.mcrypt().start(options).update(ciphertext).finalize();

mcrypt.update(message)#

암호화 혹은 복호화 할 데이터를 입력 데이터로 받습니다.

  • message <Buffer> 암호화 또는 복호화 할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • Return Value: 오브젝트 자신을 리턴합니다.
var algo_name = "AES-256";
var params = {
    key: key,
  iv: initial_vector,
  mode: "CBC",
  isEncryption: true,
  padding: "pkcs7"
};
var message1 = "This is the Message!";
var message2 = " How are you?";

var cipher = new jCastle.mcrypt(algo_name);
cipher.start(params);
cipher.update(message1);
cipher.update(message2);
var ciphertext = cipher.finalize();

  • Return Value: 암호화 또는 복호화 한 결과로 <Buffer> 데이터를 리턴합니다.

mcrypt.encrypt(message, options) #

주어진 데이터를 가지고 암호화 과정을 진행하고 그 결과 값을 리턴합니다.

  • message <Buffer> 암호화할 입력 데이터. 만일 <string>으로 주어지면 자동으로 <Buffer>로 변환됩니다.
  • options <Object> 암호화에 필요한 변수들을 담은 오브젝트로 mcrypt.start의 options 변수를 참조하세요.

var algo_name = "AES-256";
var message = "This is the Message!";
var options = {
  algoName: algo_name,
  key: key,
  iv: initial_vector,
  mode: "CBC",
  isEncryption: true
  padding: "pkcs7"
};

var ciphertext = new jCastle.mcrypt().encrypt(message, options);

options.isEncryption = false;
var plaintext = jCastle.mcrypt.decrypt(ciphertext, options);
